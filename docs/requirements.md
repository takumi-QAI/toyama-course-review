## 1. プロジェクト概要

### 1.1 目的

富山大学の在学生・卒業生が授業の口コミ・楽単情報・教科書お譲り情報を共有できる非公式プラットフォームを構築・運営する。

### 1.2 背景

- 富山大学には公式の授業評価システムが存在しない
- 学生がシラバスだけでは授業の難易度・雰囲気を把握しにくい
- 先輩から後輩への教科書の引き継ぎ手段が限られている

### 1.3 ターゲットユーザー

| ユーザー | 利用目的 |
|---------|---------|
| 在学生（一般） | 授業選択の参考・口コミ投稿・教科書購入 |
| 在学生（上位学年） | 教科書出品・後輩への情報提供 |
| 受験生・新入生 | 授業・単位の概況把握 |
| サイト管理者 | コンテンツ管理・統計確認 |

## 2. システム全体構成

```
[ユーザー]
    │
    ▼
[Vercel] ── Next.js 14 App Router (TypeScript)
    │           ├── フロントエンド (React)
    │           └── APIルート (Next.js API)
```

## 3. 機能要件

### 3.1 授業情報

#### 3.1.1 授業データの取得・更新

| 要件ID | 要件内容 | 優先度 |
|--------|---------|--------|
| C-01 | 富山大学シラバスシステム（CampusSquare）から全授業データをクローリングで取得できること | 高 |
| C-02 | 年次更新時に既存の口コミデータを保持したまま授業情報を更新できること（syllabusCode でマッチング） | 高 |
| C-03 | 取得できる授業情報：授業名・担当教員・単位数・対象学年・学期・授業区分・学科・時間割コード | 高 |
| C-04 | 複合学期（第3・第4ターム等）を適切に解析・保存できること | 中 |
| C-05 | 新設授業は自動追加、廃止授業は削除せず保持すること | 中 |

#### 3.1.2 授業一覧・検索

| 要件ID | 要件内容 | 優先度 |
|--------|---------|--------|
| S-01 | 授業名・担当教員名でキーワード検索できること（大文字小文字区別なし） | 高 |
| S-02 | 学部・学科・学期・学年で絞り込みフィルターが使えること | 高 |
| S-03 | 学部を選択すると、その学部内の学科がドロップダウンに表示されること | 中 |
| S-04 | 学期の選択肢はDBに登録されている実際の値を動的に表示すること | 中 |
| S-05 | 30件/ページのページネーションを実装すること | 高 |
| S-06 | 学年の選択肢は1〜6年生まで表示すること | 低 |
| S-07 | 名前順・楽単度順・口コミ数順・新着順で並べ替えできること | 中 |

#### 3.1.3 授業詳細

| 要件ID | 要件内容 | 優先度 |
|--------|---------|--------|
| D-01 | 授業の基本情報（学部・学科・学期・学年・単位数・授業区分）を表示すること | 高 |
| D-02 | 平均楽単度スコアと口コミ件数を表示すること | 高 |
| D-03 | 富山大学公式シラバスへの外部リンクボタンを表示すること（syllabusCode・syllabusJscd が設定されている場合） | 中 |
| D-04 | 教科書お譲りページへのリンクを表示すること | 中 |
| D-05 | ログイン済みユーザーが授業区分の訂正を提案できること | 低 |

### 3.2 口コミ機能

| 要件ID | 要件内容 | 優先度 |
|--------|---------|--------|
| R-01 | ログイン済みユーザーが楽単度（★1〜★5）と口コミ文章を投稿できること | 高 |
| R-02 | 1ユーザー1授業につき口コミは1件のみ投稿可能とすること | 高 |
| R-03 | 口コミには投稿者名・投稿日時・楽単度ラベルを表示すること | 中 |
| R-04 | 口コミは「役に立った」数（降順）→ 新着順で表示すること | 中 |
| R-05 | ログイン済みユーザーが口コミに「役に立った」を付けられること（1人1回、トグル式） | 中 |

### 3.3 AI 口コミ要約

| 要件ID | 要件内容 | 優先度 |
|--------|---------|--------|

| AI-03 | 生成された要約は24時間 DB にキャッシュすること | 中 |

## 4. 非機能要件

### 4.1 パフォーマンス

| 要件 | 目標値 |
|------|-------|
| 授業一覧 API レスポンス | 1秒以内（30件・ページネーション） |
| 授業詳細ページ初期表示 | 2秒以内 |
| シラバスクローリング（全学部） | 20分以内 |

### 4.2 セキュリティ

- パスワードは bcrypt（コスト係数10）でハッシュ化
- SQL インジェクション対策：Prisma ORM のパラメータ化クエリを使用
- XSS 対策：React の自動エスケープ
- CSRF 対策：NextAuth の組み込み保護
- 管理者 API は ADMIN_EMAIL による認証チェック
- レートリミット：DB ベースのスライディングウィンドウ（登録 5回/時、お問い合わせ 3回/時）

### 4.3 可用性

- Vercel の無料プランで稼働（SLA 99.99%）
- Neon の無料プランで稼働（オートスケール対応）
- ダウンタイム目標：月1時間以内

### 4.4 スケーラビリティ

- ページネーション実装により大量データに対応
- レビュー集計は `groupBy` による一括クエリで N+1 問題を回避
- AI 要約は24時間キャッシュで API 呼び出しを削減

## 5. データモデル

### 5.1 ER 図（概要）

```
User ─── Review ─── Course ─── Faculty
  └────── Textbook ──┘
  └────── CourseContribution ── Course

ContactMessage（独立）
PageView（独立）
```

### 5.2 テーブル定義

#### User（ユーザー）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| email | TEXT UNIQUE | メールアドレス |
| name | TEXT | ニックネーム |

| createdAt | TIMESTAMP | 登録日時 |

#### Faculty（学部）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| name | TEXT UNIQUE | 学部名 |

#### Course（授業）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| name | TEXT | 授業名 |
| instructor | TEXT | 担当教員名 |
| facultyId | TEXT FK | 所属学部 |
| year | INT | 対象学年（最若学年） |
| semester | TEXT | 学期（前期/後期/第Nターム/通年 等） |
| credits | INT | 単位数 |
| courseType | TEXT | 授業区分（必修/選択必修/選択） |
| syllabusCode | TEXT | 時間割コード（CampusSquare） |
| syllabusJscd | TEXT | 学部コード（シラバスURL生成用） |
| syllabusYear | INT | 最終クロール年度 |
| department | TEXT | 学科名（該当する場合） |
| summary | TEXT | AI 要約テキスト |
| summaryAt | TIMESTAMP | AI 要約生成日時 |
| createdAt | TIMESTAMP | 登録日時 |

#### Review（口コミ）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| content | TEXT | 口コミ本文（最大1000字） |
| easyScore | INT | 楽単度（1〜5） |
| userId | TEXT FK | 投稿者 |
| courseId | TEXT FK | 対象授業 |
| createdAt | TIMESTAMP | 投稿日時 |

#### Textbook（教科書）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |

| author | TEXT | 著者（任意） |
| isbn | TEXT | ISBN（任意） |
| price | INT | 価格（円） |
| condition | TEXT | 状態（美品/良好/普通/難あり） |
| description | TEXT | 備考（任意） |
| status | TEXT | 状態（available/sold） |
| contact | TEXT | 連絡先メール |
| sellerId | TEXT FK | 出品者 |
| courseId | TEXT FK | 対象授業 |
| createdAt | TIMESTAMP | 出品日時 |

#### ContactMessage（お問い合わせ）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| name | TEXT | 送信者名 |
| email | TEXT | 送信者メール |
| content | TEXT | 本文（最大2000字） |
| isRead | BOOL | 既読フラグ |
| createdAt | TIMESTAMP | 送信日時 |

#### PageView（ページビュー）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| path | TEXT | アクセスパス（例: /courses/xxx） |
| createdAt | TIMESTAMP | アクセス日時 |

#### CourseContribution（授業区分の提案）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | cuid() |
| courseId | TEXT FK | 対象授業 |

| field | TEXT | 変更フィールド名（例: courseType） |

| createdAt | TIMESTAMP | 提案日時 |
